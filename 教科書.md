# 教科書

## 第1章: AI時代のエンジニア戦略

### AIツールの活用法
Claude Codeには2つの使い方がある。

1. **教師として** — 概念の説明、コード解説、段階的な学習ガイド
2. **ペアプロ相手として** — コードレビュー、一緒にコードを読む、リファクタリング

ただしAIには教師としての弱点がある:
- 生徒の理解度を正確に把握できない（→ 理解度テストの仕組みで補う）
- ハルシネーション（誤った情報を自信を持って伝える可能性がある）
- 聞かれないことは教えない
- 「わかったつもり」を見抜けない

**重要**: これらの弱点は仕組みで補える。ツールの弱点を設計で克服する発想自体がエンジニアリング。

### 育成の3本柱
AI時代にエンジニアが伸ばすべきスキル:

1. **設計力** — AIはコードを書けるが、何を作るか・どう構成するかの判断は人間が担う
2. **問題解決力** — AIに「何を聞くか」を判断できる力。仮説を立ててから聞く習慣
3. **AI協働スキル** — プロンプト精度、AI出力の検証、ワークフローの型化

### AIの出力との向き合い方
AIの出力を鵜呑みにしてはいけない理由:
- AIも間違う（ハルシネーション）
- 自分で検証する力がつかないと、間違いに気づけないまま進んでしまう
- **根拠を問う習慣**が成長の鍵

## 第2章: 基礎理論 — 数値表現

### 2進数
コンピュータは電気のON(1)/OFF(0)で動くため、全てのデータを0と1で表す。

**10進数→2進数**: 2で割り続けて余りを下から読む
**2進数→10進数**: 各桁に2の累乗をかけて足す（右端が2⁰）

```
2⁰=1  2¹=2  2²=4  2³=8  2⁴=16  2⁵=32  2⁶=64  2⁷=128  2⁸=256
```

### 8進数・16進数
2進数は長くなるため、人間が読みやすくする表記法。コンピュータ内部は常に2進数。

- **8進数**: 2進数を右から**3桁ずつ**区切る（各桁の重み: 4, 2, 1）
- **16進数**: 2進数を右から**4桁ずつ**区切る（各桁の重み: 8, 4, 2, 1）
- 16進数で10以上: 10=A, 11=B, 12=C, 13=D, 14=E, 15=F
- 変換は常に**2進数を経由**するのが鉄則（10進数経由は遠回り）

実務での用途:
- 16進数 → 色コード(#FF0000)、メモリアドレス
- 8進数 → Linuxファイル権限(chmod 755)
