# 教科書

## 第1章: AI時代のエンジニア戦略

### AIツールの活用法
Claude Codeには2つの使い方がある。

1. **教師として** — 概念の説明、コード解説、段階的な学習ガイド
2. **ペアプロ相手として** — コードレビュー、一緒にコードを読む、リファクタリング

ただしAIには教師としての弱点がある:
- 生徒の理解度を正確に把握できない（→ 理解度テストの仕組みで補う）
- ハルシネーション（誤った情報を自信を持って伝える可能性がある）
- 聞かれないことは教えない
- 「わかったつもり」を見抜けない

**重要**: これらの弱点は仕組みで補える。ツールの弱点を設計で克服する発想自体がエンジニアリング。

### 育成の3本柱
AI時代にエンジニアが伸ばすべきスキル:

1. **設計力** — AIはコードを書けるが、何を作るか・どう構成するかの判断は人間が担う
2. **問題解決力** — AIに「何を聞くか」を判断できる力。仮説を立ててから聞く習慣
3. **AI協働スキル** — プロンプト精度、AI出力の検証、ワークフローの型化

### AIの出力との向き合い方
AIの出力を鵜呑みにしてはいけない理由:
- AIも間違う（ハルシネーション）
- 自分で検証する力がつかないと、間違いに気づけないまま進んでしまう
- **根拠を問う習慣**が成長の鍵

## 第2章: 基礎理論 — 数値表現

### 2進数
コンピュータは電気のON(1)/OFF(0)で動くため、全てのデータを0と1で表す。

**10進数→2進数**: 2で割り続けて余りを下から読む
**2進数→10進数**: 各桁に2の累乗をかけて足す（右端が2⁰）

```
2⁰=1  2¹=2  2²=4  2³=8  2⁴=16  2⁵=32  2⁶=64  2⁷=128  2⁸=256
```

### 8進数・16進数
2進数は長くなるため、人間が読みやすくする表記法。コンピュータ内部は常に2進数。

- **8進数**: 2進数を右から**3桁ずつ**区切る（各桁の重み: 4, 2, 1）
- **16進数**: 2進数を右から**4桁ずつ**区切る（各桁の重み: 8, 4, 2, 1）
- 16進数で10以上: 10=A, 11=B, 12=C, 13=D, 14=E, 15=F
- 変換は常に**2進数を経由**するのが鉄則（10進数経由は遠回り）

実務での用途:
- 16進数 → 色コード(#FF0000)、メモリアドレス
- 8進数 → Linuxファイル権限(chmod 755)

### 補数

#### なぜ補数が必要か
コンピュータには**引き算の回路がない**。補数を使うことで、足し算だけで引き算を実現している。これがコンピュータ内部で負の数を扱う基本原理。

#### 2種類の補数
n進数には2種類の補数がある。

| 補数の種類 | 意味 | 10進数での例 |
|---|---|---|
| **nの補数**（基数の補数） | ある数をnのべき乗にするために足す数 | 396の10の補数 → 604（396+604=1000） |
| **(n-1)の補数**（減基数の補数） | 各桁を最大値にするために足す数 | 396の9の補数 → 603（396+603=999） |

**関係**: nの補数 = (n-1)の補数 + 1

#### 2進数での補数

| 補数 | 求め方 | 例（8ビット: 01010011） |
|---|---|---|
| **1の補数** | 全ビットを反転（0↔1） | 10101100 |
| **2の補数** | 1の補数 + 1 | 10101101 |

#### 2の補数 = 負の数の表現
コンピュータは**2の補数**で負の数を表す。

例: 8ビットで -5 を表す手順
```
1. +5 を2進数にする   → 00000101
2. ビット反転（1の補数）→ 11111010
3. +1する（2の補数）   → 11111011  ← これが -5
```

検算: 5 + (-5) = 0 になるか確認
```
  00000101  (+5)
+ 11111011  (-5)
----------
 100000000  ← 9ビット目はあふれて消える → 00000000 = 0 ✅
```

#### 符号ビット
最上位ビット（先頭ビット）が符号を表す。

- **0**xxxxxxx → 正の数（8ビットなら 0〜127）
- **1**xxxxxxx → 負の数（8ビットなら -128〜-1）

#### 補数を使った引き算の例
7 - 3 を8ビットの足し算で計算する:
```
7 - 3  =  7 + (-3)

-3 の2の補数: 00000011 → 11111100 → 11111101

  00000111  (+7)
+ 11111101  (-3)
----------
 100000100  ← あふれを無視 → 00000100 = 4 ✅
```

### 論理演算

#### 4つの基本演算

**AND（論理積）** — 両方1のときだけ1
```
0 AND 0 = 0    0 AND 1 = 0    1 AND 0 = 0    1 AND 1 = 1
```

**OR（論理和）** — どちらか1つでも1なら1
```
0 OR 0 = 0     0 OR 1 = 1     1 OR 0 = 1     1 OR 1 = 1
```

**NOT（否定）** — ひっくり返す
```
NOT 0 = 1      NOT 1 = 0
```

**XOR（排他的論理和）** — 片方だけ1のとき1（ひねくれもの）
```
0 XOR 0 = 0    0 XOR 1 = 1    1 XOR 0 = 1    1 XOR 1 = 0
```
※ ORとの違い: 1 OR 1 = 1 だが、1 XOR 1 = 0

#### 複数ビットでの演算
各ビットごとに独立して計算する。
```
    10110100        10110100        10110100
AND 11001110    OR  11001110    XOR 11001110
----------      ----------      ----------
    10000100        11111110        01111010
```

#### 実務での用途

| 演算 | 用途 |
|---|---|
| AND | 特定のビットだけ取り出す（マスク処理） |
| OR | 特定のビットを1にする（フラグ設定） |
| NOT | ビット反転 = 1の補数（補数と同じ！） |
| XOR | 値の入れ替え、暗号化、差分チェック |

#### 他の学習テーマとのつながり
- **NOT = 1の補数**: 補数で学んだ「ひっくり返す」はNOT演算そのもの
- **NOT + 1 = 2の補数**: つまり2の補数 = NOT演算 + 加算で求められる
