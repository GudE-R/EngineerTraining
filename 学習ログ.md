### 2進数・8進数・16進数
- 10進数→2進数: 2で割り続けて余りを下から読む
- 2進数→10進数: 各桁に2の累乗をかけて足す
- 2進数→8進数: 右から3桁ずつ区切って各グループを変換
- 2進数→16進数: 右から4桁ずつ区切って各グループを変換
- 8進数⇔16進数: 2進数を経由して変換する
- 2の累乗は暗記必須: 1, 2, 4, 8, 16, 32, 64, 128, 256

### 補数
- 補数の目的: コンピュータには引き算の回路がないため、足し算だけで引き算を実現する仕組み
- 1の補数: 全ビットを反転（0↔1）
- 2の補数: 1の補数 + 1
- コンピュータは2の補数で負の数を表現する
- 符号ビット: 最上位ビットが0なら正、1なら負
- n の補数 = (n-1)の補数 + 1 という関係が成り立つ

### 論理演算
- AND（論理積）: 両方1のときだけ1
- OR（論理和）: どちらか1つでも1なら1
- NOT（否定）: ビット反転（0↔1）
- XOR（排他的論理和）: 片方だけ1のとき1（両方1は0 ← ORとの違い）
- 複数ビットの演算は各ビットごとに独立して計算する
- 実務用途: AND=マスク処理、OR=フラグ設定、NOT=ビット反転、XOR=値の入れ替え・暗号化

### データ構造
- 配列: 番号（インデックス）で即アクセス。途中への挿入は苦手
- リスト（連結リスト）: 挿入・削除が得意。番号で直接取り出せない
- スタック: 後入れ先出し（LIFO）。push/popの2操作。例: ブラウザの戻る、Ctrl+Z
- キュー: 先入れ先出し（FIFO）。enqueue/dequeueの2操作。例: 印刷待ち
- 木: 階層構造。ルート・ノード・リーフ。例: フォルダ構造、HTML
- データ構造の選び方 = そのデータに対して一番多い操作は何かで決まる

### プログラミング基本文法（言語共通）
- 変数: データに名前をつける箱。型には整数・小数・文字列・真偽値・nullがある
- 演算子: 算術(+ - * / %)、比較(== != < > <= >=)、論理(&& || !)
- 条件分岐: if / else if / else で処理を分ける
- ループ: for（回数指定）、while（条件指定）、forEach（配列の各要素に処理）
- 関数: 処理をまとめて名前をつける。引数（入力）と戻り値（出力）を持つ
- 配列操作: インデックスで取得、push/filter/map
- オブジェクト: 関連するデータをキーと値のペアでまとめる
- === と == の違い: ===は型も値も一致で判定、==は型変換して比較（バグの元）。実務では===を使う（JS/TS特有）

### アルゴリズム
- アルゴリズム = 問題を解くための手順。手順が違えば速さが変わる
- 線形探索: 先頭から順番に探す。ソート不要。O(n)
- 二分探索: 真ん中で半分に切って絞り込む。ソート済み必須。O(log n)
- バブルソート: 隣同士を比べて入れ替え。O(n²)で遅い。教科書向け
- 選択ソート: 最小値を見つけて先頭に置く。O(n²)で遅い
- クイックソート: 基準で2グループに分けて再帰。O(n log n)で実務向き
- 再帰: 自分自身を呼んで問題を小さくしていく。止まる条件が必須
- 計算量: O(1) < O(log n) < O(n) < O(n log n) < O(n²) の順に遅くなる

### ネットワーク基礎
- クライアント（リクエストする側）とサーバー（返す側）の関係
- HTTP = 通信のルール（プロトコル）
- HTTPメソッド: GET=取得、POST=作成、PUT/PATCH=更新、DELETE=削除
- ステータスコード: 2xx=成功、4xx=クライアントミス、5xx=サーバーエラー
- JSON = サーバーとの共通データ形式。TSのオブジェクトとほぼ同じ
- API = サーバーが用意した窓口（REST API: URL+メソッドの組み合わせ）
- トークン認証: ログイン→トークン発行→以降のリクエストにトークンを付ける（入場チケット）
- HTTPS = HTTPを暗号化して安全にしたもの
